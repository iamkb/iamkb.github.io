Docker引擎连接问题

发表于 2018-05-11 | 分类于 技术笔记 ， Docker | 阅读次数

环境

  • Docker Toolbox on Windows全家桶
  • Win7 x64 旗舰版

起因

在使用docker-maven-plugin插件将项目打包成docker镜像时，遇到了如下问题：

Failed to execute goal com.spotify:docker-maven-plugin:0.4.12:build (default-cli) on project demo: Exception caught:
Timeout: GET https://192.168.99.101:2376/version: com.spotify.docker.client.shaded.javax.ws.rs.ProcessingException: org.apache.http.conn.ConnectTimeoutException: Connect to 192.168.99.101:2376 [/192.168.99.101] failed: connect timed out -> [Help 1]

分析

显然，docker连接的时候炸了，但是我的docker还是能正常使用，说明是哪里配置又不对劲了！

解决过程

1.使用全家桶之Kitematic (Alpha)连接下docker引擎，出现

Cannot connect to the Docker Engine. Either the
VM is not responding or the connection may be
blocked (VPN or Proxy): Hostname/IP doesn’t
match certificate’s altnames: “IP: 192.168.99.100 is
not in the cert’s list: 192.168.99.101”

2.查看环境变量，发现DOCKER_HOST不大对劲，地址为192.168.99.101，但是应该是100才对，顺手改掉，

然后进入DOCKER_CERT_PATH,看到config.json文件的ip地址也不对劲，顺手也改掉

3.再次使用Kitematic尝试连接，结果失败！docker-maven-plugin打包也还是不行。进行谷歌，搜到类似：
Start-up stops at 99% - IP: 192.168.99.100 is not in the cert’s list:192.168.99.101

应该是认证相关问题，执行以下命令

4.再次使用Kitematic连接，没问题，maven插件也可以了

出现这个问题是因为当时装好全家桶后，使用了VirtualBox已经开了一台machine，然后才起的docker，导致docker的虚拟机是在101，后来删除了之前那台，docker回到了100。ip的变化导致的问题！
